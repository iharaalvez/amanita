# UX + BA Roadmap

This roadmap turns the latest UX/BA review into implementation-ready work. The focus is to make every progress number trustworthy, keep mobile flows complete, and make the app feel like a polished Living Pokedex workspace.

## Product Principles

- Living Dex means one of each base species.
- HOME can include regional and cosmetic forms, but those counts must not redefine Living Dex completion.
- Games are user-owned or user-tracked titles, not just a filter.
- Shiny tracking is a parallel goal, not a replacement for normal completion.
- Empty states should always offer a next action.

## Phase 1 - Fix Trust In Progress Numbers

### 1. Standardize Living Dex Counts

**Status:** Implemented.

Living Dex completion counts base species only. HOME can show form-inclusive slot counts, but those are labeled separately.

### 2. Clarify Shiny Charm Progress Copy

**Status:** Implemented.

Incomplete eligible games show remaining Shiny Charm progress, while completed eligible games show Charm unlocked.

## Phase 2 - Make Stats Feel Like A Dashboard

### 3. Restyle Stats For The Current App Shell

**Status:** Implemented.

Stats now uses the same dark dashboard language as Games and HOME, with clearer card hierarchy and scan-friendly sections.

### 4. Add Actionable Empty States

**Status:** Implemented.

Empty states now point users toward Pokemon details, Pokedex work, or HOME/game setup instead of ending with passive copy.

## Phase 3 - Improve HOME Density Controls

### 5. Split Shiny HOME Display Modes

**Status:** Implemented.

HOME supports Normal, Shiny, and Pair display modes. The selected mode persists locally.

### 6. Improve HOME Filter Persistence While Scrolling

**Status:** Implemented.

HOME filters, search, forms, and display mode remain reachable during long scroll sessions.

## Phase 4 - Mature The Games Hub

### 7. Add Generation Filtering Or Grouping

**Status:** Implemented.

The Games hub groups the add-more catalog by generation and offers generation filter chips.

### 8. Add Per-Game Summary Signals

**Status:** Implemented.

Game cards show registered count, missing count, percent, and Shiny Charm remaining/unlocked signals when curated dex totals are available.

## Phase 5 - Future Product Expansion

### 9. Location And Encounter Guidance

**Status:** Implemented.

Pokemon details merge PokeAPI encounter data with stored/curated encounter data, and the per-game Hunt Guide surfaces missing targets.

### 10. Pokedex Reference Depth

**Status:** Proposed.

Candidate work:

- Type chart.
- Moves per Pokemon.
- Evolution and breeding guidance refinements.
- Location-per-game browser.
